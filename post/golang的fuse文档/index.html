bazil.org/fuse 是一个用于编写FUSE用户态空间文件系统的Go库。

bazil.org/fuse – Filesystems in Go

它是内核-用户态空间通信协议的一个从头开始的实现，并且不使用来自名为FUSE的项目的C库。bazil.org/fuse完全拥抱安全且易于编程。

以下是如何开始:

go get bazil.org/fuse

网站: http://bazil.org/fuse/

Github仓库: https://github.com/bazil/fuse

API文档: http://godoc.org/bazil.org/fuse

非常感谢Russ Cox，这个项目是基于他的fuse代码库开始的。

包fuse支持在Linux和FreeBSD上写入fuse文件系统。

编写FUSE文件系统有两种方法。第一个是讲低级消息协议，使用ReadRequest从Conn读取，使用各种response方法写入。这种方法最接近与内核的实际交互，在协议翻译等上下文中可能是最简单的方法。

合成文件系统的服务器倾向于共享由第二种方法抽象出来的公共簿记，即调用fs。服务使用FS*(文件系统)、Node*(文件或目录)和Handle*(打开的文件或目录)接口中的服务方法的实现来服务FUSE协议。可以编写的此类方法数量惊人，但很少是必需的。文档中描述了这些接口的具体方法。

hellofs子目录包含fs的一个简单示例。服务的方法。

服务方法

FS、Node和Handle接口的必需和可选方法具有通用形式

Op(ctx context.Context, req *OpRequest, resp *OpResponse) error

其中Op是FUSE操作的名称。Op从req中读取请求参数并将结果写入resp。如果操作的唯一结果是错误结果，则忽略了resp参数。

多个goroutine可以同时调用服务方法;被调用的方法负责适当的同步(线程安全)。

操作不能保留请求或响应，包括任何[]字节字段，如WriteRequest。数据或SetxattrRequest.Xattr。

错误

操作可能会返回错误。FUSE接口只能与文件系统客户端用POSIX errno错误号通信，该消息对文件系统客户端是不可见的。返回的错误可以实现ErrorNumber来控制返回的errno。如果没有ErrorNumber，则返回一个通用的errno (EIO)。

错误消息将作为响应的一部分在调试日志中可见。

中断操作

在某些文件系统中，某些操作可能会花费不确定的时间。例如，等待网络消息的Read或匹配的Write可能无限期地等待。如果请求被取消且不再需要，则上下文将被取消。阻塞操作应该选择从ctx.Done()接收的内容，如果接收成功(意味着通道关闭)，则尝试提前终止该操作。若要指示操作因中止而失败，请返回syscall.EINTR。

如果一个操作不会阻塞无限长的时间，那么就没有必要支持取消。

身份验证

所有请求类型都嵌入Header，这意味着该方法可以检查req.Pid、req.Uid和req.Gid，以实现权限检查。内核FUSE层通常阻止其他用户访问FUSE文件系统(要改变这一点，请参阅AllowOther)，但不强制使用访问模式(要改变这一点，请参阅DefaultPermissions)。

挂载操作

可以通过将MountOption值传递给Mount来更改挂载文件系统的行为和元数据。

func (*Conn) InvalidateEntry

func (c *Conn) InvalidateEntry(parent NodeID, name string) error

InvalidateEntry让名为name的目录条目的内核缓存失效,失效的目录条目由父节点的NodeID和目录条目的name确定。

内核可能缓存目录列表，也可能不缓存。要使它们无效，请使用InvalidateNode使目录的所有数据无效。(截至2015-06,Linux FUSE不缓存目录清单。)

如果内核当前没有缓存节点，则返回ErrNotCached。

func (*Conn) InvalidateNode

func (c *Conn) InvalidateNode(nodeID NodeID, off int64, size int64) error

InvalidateNode使一个节点的属性和节点关联的指定范围的数据在内核中的缓存失效。

offset为0,size为-1表示所有数据。要仅使属性无效，请给出偏移量0和大小0。

如果内核当前没有缓存节点，则返回ErrNotCached。

type LockRequest

type LockRequest struct {
	Header
	Handle HandleID
	// LockOwner is a unique identifier for the originating client, to
	// identify locks.
	LockOwner LockOwner
	Lock      FileLock
	LockFlags LockFlags
}

LockRequest请求尝试获取节点上的字节范围锁。响应应该是立即的，而不是等待获取锁。

解锁就可以

- explicit with UnlockRequest
- for flock: implicit on final close (ReleaseRequest.ReleaseFlags
  has ReleaseFlockUnlock set)
- for POSIX locks: implicit on any close (FlushRequest)
- for Open File Description locks: implicit on final close
  (no LockOwner observed as of 2020-04)

请参阅LockFlags了解正在请求的锁的类型。(截至2020-04年，打开文件描述符锁与POSIX没有区别。这意味着使用这些锁的程序在关闭基于fuse的分布式文件系统上的fd时很可能会行为不端，因为文件系统只知道遵循POSIX锁定规则并过早释放全局锁。)

flock (BSD)和POSIX (fcntl F_SETLK)锁之间的大多数其他区别只与调用方有关，与文件系统无关。FUSE总是看到范围，并将flock整个文件锁视为对最大字节范围的请求。文件系统也应该这样做，因为这为linux本地的Open文件描述锁提供了向前兼容路径。

要在FUSE中启用锁定事件，请将LockingFlock()和/或LockingPOSIX()传递给Mount。

也看到LockWaitRequest。

拥塞阈值设置未完成的后台FUSE请求的数量，超过该数量内核认为文件系统拥塞。这可能有助于处理请求延迟。

func DaemonTimeout(name string) MountOption

DaemonTimeout设置在声明FUSE挂载死亡之前，请求和应答之间的时间(以秒为单位)。

FreeBSD。其他人忽略了这个选项。

func DefaultPermissions

func DefaultPermissions() MountOption

DefaultPermissions使内核基于文件模式强制访问控制(如chmod)。

如果没有这个选项，Node自己决定什么是允许的，什么是不允许的。这通常是可以的，因为没有AllowOther，其他用户无法访问FUSE文件系统。

FreeBSD忽略此选项。

func FSName

func FSName(name string) MountOption

FSName设置在挂载的文件系统列表中可见的文件系统名称(也称为源)。

FreeBSD忽略此选项。

func LockingFlock

func LockingFlock() MountOption

LockingFlock支持基于flock (BSD)的锁定。这对于具有全局锁定的分布式文件系统非常有用。如果没有这个，内核将自动管理本地锁定。

func LockingPOSIX

func LockingPOSIX() MountOption

LockingPOSIX支持基于flock (BSD)的锁定。这对于具有全局锁定的分布式文件系统非常有用。如果没有这个，内核将自动管理本地锁定。

要注意，POSIX锁对调用者来说是一个有缺陷的API，具有不直观的行为。

func MaxBackground

func MaxBackground(n uint16) MountOption

MaxBackground设置内核将在后台提交的FUSE请求的最大数量。当不需要立即回答时，使用后台请求。这可能有助于处理请求延迟。

在Linux上，这可以通过/sys/fs/fuse/connections/CONN/max_background动态调整

func MaxReadahead

func MaxReadahead(n uint32) MountOption

MaxReadahead设置顺序读取时可以预取的字节数。内核可以强制设置一个比这个更低的最大值。

这个设置使得内核执行投机性读取，而不是来自任何客户端进程。这通常会极大地提高读性能。

func ReadOnly

func ReadOnly() MountOption

ReadOnly使挂载为只读。

func Subtype

Subtype设置挂载的子类型。主要类型总是“fuse”。挂载的文件系统列表中的类型看起来像' fuse.foo ‘。
